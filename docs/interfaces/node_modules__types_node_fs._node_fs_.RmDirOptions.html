Optional maxRetries

maxRetries: number
  • Defined in node_modules/@types/node/fs.d.ts:1391

If an EBUSY, EMFILE, ENFILE, ENOTEMPTY, or EPERM error is encountered, Node.js will retry the operation with a linear backoff wait of retryDelay ms longer on each try. This option represents the number of retries. This option is ignored if the recursive option is not true.

default

0

Optional recursive

recursive: boolean
  • Defined in node_modules/@types/node/fs.d.ts:1401
deprecated

since v14.14.0 In future versions of Node.js and will trigger a warning fs.rmdir(path, { recursive: true }) will throw if path does not exist or is a file. Use fs.rm(path, { recursive: true, force: true }) instead.

If true, perform a recursive directory removal. In recursive mode soperations are retried on failure.

default

false

Optional retryDelay

retryDelay: number
  • Defined in node_modules/@types/node/fs.d.ts:1407

The amount of time in milliseconds to wait between retries. This option is ignored if the recursive option is not true.

100
